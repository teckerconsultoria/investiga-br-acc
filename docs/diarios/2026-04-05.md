## Decisoes Tecnicas
- **Hibrido REST + WebSocket**: REST para dados estaticos (fontes, config) e WebSocket para execucao em tempo real — melhor separacao de responsabilidades que misturar tudo em um unico canal
- **Sem Supabase**: stack existente (Neo4j + JWT + cookies) ja resolve auth; adicionar PostgreSQL seria overkill para o escopo atual
- **Docker-in-Docker via socket**: montado `/var/run/docker.sock` no container API para permitir execucao de ETL via `docker compose` — evita necessidade de Docker daemon separado
- **Volume `/app/host`**: mapeamento do repo raiz no container API para que `docker compose` encontre o `docker-compose.yml` e o profile ETL funcione corretamente
- **3 modos de execucao no frontend**: Individual (fonte especifica), Core (7 fontes essenciais), Bootstrap (todas) — resolve o problema de UX onde o botao "Executar" nao habilitava para fontes sem `pipeline_id`

## Bugs Corrigidos
- **Fontes nao listadas**: CSV `source_registry_br_v1.csv` nao estava disponivel no container da API (so `src/` era copiado no Dockerfile) → montado via volume `./docs:/app/docs` + env `BRACC_SOURCE_REGISTRY_PATH`
- **Config "erro ao carregar"**: `config/bootstrap_all_contract.yml` tambem nao montado → adicionado volume `./config:/app/config`
- **WebSocket falhando**: nginx nao fazia proxy de WebSocket → adicionada location `/api/v1/admin/ws` com headers `Upgrade` e `Connection`
- **Botao Executar nao habilitava**: select incluia fontes sem `pipeline_id` → filtrado para `implementation_state === "implemented"`
- **ETL "No such file or directory"**: container API sem Docker CLI → instalado `docker-ce-cli` + `docker-compose-plugin` no Dockerfile, montado `docker.sock`
- **`_get_repo_root()` retornava `/`**: no container, `parents[4]` do path do modulo era `/` → fallback para `/app/host` quando montado
